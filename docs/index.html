The problem

Signal exists. Someone still has to find it.

A credential leak posted to GitHub. A jailbreak walkthrough on YouTube. A frustrated bug report on Hacker News. None of it arrives in an inbox — it has to be found, and by the time it's found by chance, it's often already spreading. Claude Radar treats “public mention of Claude” as a real signal worth polling for, the same way an ops team treats latency or error rate.

How it works

Five stages, each one real

Every stage below is a working code path, not a diagram of an idea.

01

Collect

Poll Reddit, YouTube, Hacker News, Stack Overflow, and GitHub for Claude-related terms on a schedule — X and Mastodon are wired in and ready to switch on.

02

Classify

Claude reads every post: is this a real pain point? which of eight categories? what severity? which model is implicated?

03

Score

Engagement velocity, not raw volume, decides what becomes an alert — tunable per platform, per category, and per model.

04

Cluster

Related posts across platforms collapse into one root-cause incident, so five reports of the same bug read as one thing, not five.

05

Respond

A claim-and-Kanban workflow: land a card in a working column and Claude drafts a Course of Action grounded in that category's real playbook — an editable checklist, not a suggestion nobody tracks.

Inside the tool

What a shift actually looks like

Four screens from the running dashboard — no mockups.

Home · footprint graph

Every tracked post, one graph

Hubs are root-cause clusters; satellites are individual posts, colored by platform and sized by velocity. A flashing red ring means an open incident nobody has acknowledged yet.

Board · personal workspace

Claim it, and Claude drafts the response

Landing a card in Acknowledged, Mitigating, or Resolved auto-generates a Course of Action grounded in that category's escalation playbook — paired with a concrete, checkable action list, not just prose.

Incident detail

The full lifecycle, in one panel

Timeline, an actionable checklist, an executive brief, and a post-incident report — every one of them written by Claude, on demand, from the same underlying facts.

Alerts · team queue

One shared queue, no double work

Everything the system has flagged, whether it's been claimed yet or not — so a team can see what's already someone's before picking it up.

What's next

What this becomes with Anthropic's resources

Everything above runs on public APIs and a single Claude API key, built entirely outside Anthropic's own infrastructure. It's a real, working prototype of the shape — not the ceiling.

Direct platform partnerships 01

Reddit, YouTube, X, and others all run real Trust & Safety and customer-safety APIs. Privileged access replaces public search endpoints with faster, more complete signal — and no rate-limit ceiling.

Wider collection at scale 02

Discord, LinkedIn, TikTok, and Threads are already wired into the source picker, waiting on infrastructure this project doesn't have. Coverage stops being a matter of which free API happens to be open.

Claude-native analysis 03

Classification, scoring, and every recommended action already run on the public Claude API. With Anthropic's own model access and internal tooling, the same pipeline gets faster and sharper — a proof of concept becomes a production one.
